## Overview

This document describes the UI for the RFT (Reinforcement Fine-Tuning) product. The product helps users create fine-tuned models from their LLM gateway traces.

**Key Concept:** Our platform is an LLM Gateway. Users route their LLM traffic through us, generating traces. This product transforms those traces into fine-tuned models.

**Design Philosophy:** The main interface is a **visual canvas** showing the pipeline as connected nodes (similar to Figma/Stitch). Records are accessed via contextual overlays from the canvas.

# Part 2: Create Dataset (Single Screen)

## Screen: Create Dataset

**Route:** `/finetune/new`  
**Purpose:** Single screen to filter traces, configure dataset, and start pipeline.

This screen adapts based on trace availability:
- **Normal state:** User has traces, can filter and select
- **Empty state:** No traces in database (edge case for new users)
- **No results state:** Filters return zero matches

### Validation Rules

Records are validated automatically. Invalid reasons include:

| Reason | Description |
|--------|-------------|
| No system message | Missing system prompt |
| No user message | No user turn in conversation |
| Empty assistant reply | Assistant response is empty |
| Tool call malformed | Invalid tool call JSON |
| Exceeds token limit | Conversation too long for training |
| Missing required field | Schema validation failed |

### Behavior

- **Auto-runs:** On initial load, after adding data, after editing records
- **Real-time:** Shows "Validating..." during checks
- **Non-blocking:** Pipeline continues with valid records only
- **Persistent:** Always visible, always up-to-date

}
